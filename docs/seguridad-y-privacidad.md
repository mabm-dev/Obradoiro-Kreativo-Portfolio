# Seguridad y privacidad

Este repositorio esta preparado para ser publico en GitHub sin exponer informacion sensible del proyecto.

## Archivos que no se incluyen

No se incluyen:

- `wp-config.php`
- Base de datos `.sql`
- Credenciales de administrador.
- Usuarios reales.
- Pedidos reales.
- Emails de clientes.
- Datos SMTP.
- Claves API.
- Licencias de plugins.
- Carpetas completas de plugins o temas comerciales.

## Por que no se sube WordPress completo

WordPress contiene muchos archivos que no son necesarios para valorar el trabajo realizado. Ademas, una instalacion completa puede incluir configuraciones privadas, rutas internas o datos sensibles.

Por eso este repositorio solo contiene:

- Codigo personalizado.
- Documentacion.
- Capturas optimizadas.
- Explicacion de plugins utilizados.

## Buenas practicas aplicadas

- Separacion entre codigo propio y herramientas externas.
- Documentacion de plugins en lugar de subir sus carpetas.
- Uso de `.gitignore`.
- Eliminacion de datos privados.
- Capturas sin informacion sensible de clientes reales.

## Recomendacion antes de publicar

Antes de subir el repositorio a GitHub, revisar que no haya:

- Correos privados.
- Contraseñas.
- Tokens.
- Archivos de copia de seguridad.
- Exportaciones de base de datos.
- Capturas con datos personales.
